# core-rules.md 작성 가이드 (이 파일은 hook이 읽지 않음)

`core-rules.md`는 `UserPromptSubmit` hook(`~/.claude/hooks/inject_core_rules.py`)이
매 프롬프트마다 읽어서 Claude의 컨텍스트에 `additionalContext`로 주입한다.
대화가 길어져도 핵심 지침이 흐려지지 않게 하는 "망각 방지" 바닥이다.

## 작성 규칙

- **사실 진술로 쓴다.** 명령조("~하지 마")로 쓰면 Claude Code의 prompt-injection
  방어가 발동해 무시될 수 있다. "이 개발자는 ~를 선호한다 / ~를 신뢰하지 않는다"
  같은 사실 문장으로 쓴다.
- **짧게 유지한다.** 매 턴 주입되므로 길면 토큰 낭비 + 효과 희석. 9000자에서 잘린다.
- **행동 규칙을 둔다.** 대화가 길어지면 모델이 흩어지는 *행동* 규칙이 여기 적합하다.
  OS·패키지매니저 같은 정적·불변 사실은 `CLAUDE.md`가 더 낫다(스크립트 없이 로드됨).

## hook이 처리하는 방식

- 맨 위 H1 제목(`# core-rules`) 줄은 주입에서 제외된다.
- `<!-- ... -->` HTML 주석은 제거된다. (단, 주석 본문 *안에* literal `-->`를 넣지 말 것.)
- 파일이 없거나 비어 있으면 아무것도 주입하지 않는다(무해).

## 편집하면 언제 반영되나

- 다음 프롬프트 제출 시점부터 자동 반영. Claude Code 재시작 불필요(파일을 매번 읽음).
